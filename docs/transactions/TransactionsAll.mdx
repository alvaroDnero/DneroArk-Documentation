# Title 


<div className="highlight-box">

Receives a list of transactions, starting from the most recent to the oldest, with pagination support.

If no parameters are provided, it returns all notifications from the most recent to the oldest. If we want to use the paginator, we can include the parameters.

Coin Statuses

- 1 = Received, pending to be collected.
- 2 = Sent, pending to be collected.
- 3 = Received and collected.
- 4 = Sent and collected.
- 5 = Sent and expired.
- 6 = Received and expired.



### Header

| Key      | Value     |
|-------------|----------|
| `Accept` | application/json |
| `Authorization` | Bearer key|


---
### Parameter

| Name       | Type      |  Description                                                                                   |
|------------|-----------|-----------------------------------------------------------------------------------------------|
| `statuses` | `int[]`   | An array of statuses to filter the transactions. If `null` or `0`, all statuses are returned. Multiple statuses can be provided. |

**Query parameters example**

{apiPath}```?statuses=1,2&pageSize=10&page=1```





| Name       | Type      |  Description                                                                                   |
|------------|-----------|-----------------------------------------------------------------------------------------------|
| `pageSize` | `int`     | The number of items to include in each page of the results. Required if using the paginator.  |
| `page`     | `int`     | The page number to retrieve, starting from 1 for the first page. Required if using the paginator. |

**Query parameters example**

{apiPath}```?pageSize=10&page=2```

| Name       | Type      |  Description                                                                                   |
|------------|-----------|-----------------------------------------------------------------------------------------------|
| `statuses` | `int[]`   | An array of statuses to filter the transactions. If `null` or `0`, all statuses are returned. Multiple statuses can be provided. |
| `pageSize` | `int`     | The number of items to include in each page of the results. Required if using the paginator.  |
| `page`     | `int`     | The page number to retrieve, starting from 1 for the first page. Required if using the paginator. |


**Query parameters example**

**{apiPath}```?statuses=1,2&pageSize=10&page=3```**


---

### JSON Response: Success (200)
| Name       | Type       |  Description                                                                                  |
|------------|------------|----------------------------------------------------------------------------------------------|
| `statuses` | `int`     | An array of statuses to filter the transactions. If `null` or `0`, all statuses are returned. Multiple statuses can be provided. |
| `pageSize` | `int`     |  The number of items to include in each page of the results.                                   |
| `page`     | `int`     |  The page number to retrieve, starting from 1 for the first page.                              |
| `id`       | `long`     |  Unique identifier of the transaction notification.                                           |
| `type`     | `string`   |  Possible values: `sent` or `received`.                                                      |
| `userName` | `string`   |  If `type` is `sent`, it’s the recipient's name; if `received`, it’s the sender's name.       |
| `date`     | `DateTime` |  The date and time when the transaction was created.                                          |
| `amount`   | `string`   |  The amount of money involved in the transaction.                                             |
